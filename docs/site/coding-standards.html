Coding Standards

Repository-level standards for implementation quality, localization discipline, test quality, and documentation quality.

General standards

  • Write code and technical documentation in English.
  • Keep changes scoped to issue requirements and avoid unrelated refactors.
  • Prefer clear names over abbreviations.
  • Keep functions cohesive and focused on a single responsibility.

Frontend standards (React)

  • Use functional components and hooks.
  • Keep route definitions centralized and predictable.
  • Avoid hardcoded user-facing strings in components.
  • Localization keys must stay aligned for en, es, and pt.

Automation standards

  • Map automated tests to acceptance criteria and user-visible behavior.
  • For i18n scope, include language switch assertions and translated-text checks.
  • Keep scenarios deterministic and CI-friendly.

Language switch requirement (strict)

  • The product supports runtime language switching through the header selector.
  • Supported locales are strictly: en, es, and pt.
  • Any UI feature touching labels/navigation/copy must include localization updates and verification.
