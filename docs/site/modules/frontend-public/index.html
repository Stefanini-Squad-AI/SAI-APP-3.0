Frontend Public Module

Purpose

The frontend-public module is the React SPA that serves the public-facing site: home, services, about, FAQ, contact, calculator, and static legal pages (Privacy Policy, Terms and Conditions, Legal Information). It does not require authentication and shares the same layout (Header, Footer) and i18n (es, en, pt).

Public routes

  • / — Home
  • /services — Services
  • /about — About Us
  • /faq — FAQ
  • /contact — Contact
  • /calculator — Calculator
  • /privacy — Privacy Policy
  • /terms — Terms and Conditions
  • /legal — Legal Information (titular, datos registrales, aviso legal)

Technical foundation

React 18, React Router, i18next, Tailwind CSS. Pages under frontend/src/pages/; layout in frontend/src/components/layout/.

Dependencies

Depends on AuthContext for optional admin link only; no API calls for static legal pages. i18n locales in frontend/src/i18n/locales/.

Reuse patterns

Static legal pages (Privacy, Terms, Legal) follow the same pattern: hero section, max-w-4xl content container, sections with h2/p, and footer links "Contáctanos" (→ /contact) and "Volver al inicio" (→ /).

Acceptance criteria examples

  • AC1: /legal returns 200 and shows the translated page title.
  • AC2: Header (desktop and mobile) shows "Información legal" link after "Contacto", pointing to /legal.
  • AC3: Footer Legal section includes an link to Legal Information pointing to /legal.
  • AC4: Legal page has hero, at least one content section, and links to Contact and Home.
  • AC5: All labels and page content use i18n keys (legalPage.*, header.legalInfo) in es, en, pt.
