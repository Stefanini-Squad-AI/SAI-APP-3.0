System Overview

Architecture map, module catalog, routes, i18n boundaries, and integration surfaces.

Architecture and domain

  • frontend/: React 18 + Vite + Tailwind CSS single-page application.
  • backend/: ASP.NET Core Web API (.NET 8) with JWT authentication and MongoDB repositories.
  • database/: MongoDB initialization and seed support for local dockerized environments.
  • automation/: quality tooling, including applied-ai-aura functional automation.

Module catalog

ModulePurposeTech
frontend-publicPublic web flows and legal/informational pagesReact, Router, i18next
frontend-adminJWT-protected administration consoleReact, protected routes
backend-apiREST API for auth and business operationsASP.NET Core, JWT, MongoDB
automation-auraFunctional automation scenarios and reportingTypeScript, Cucumber

Internationalization (strict)

  • Supported runtime locales are strictly: en, es, and pt.
  • Language switch is provided in the header selector and persisted with tco-user-language.

SAIAPP3-21 — Legal Information Page

The /legal route was introduced as part of SAIAPP3-21, adding a dedicated Legal Information page accessible from both the header navigation (after Contact) and the footer (replacing the previous /terms link for the "Aviso Legal" entry). The following changes were made across the frontend-public module:

File Change AC coverage
pages/LegalPage.jsx New component with hero section (h1), four content sections (h2 each): Legal Notice, Limitation of Liability, Intellectual Property, Jurisdiction & Contact. AC1, AC3, AC5
App.jsx New route /legal registered inside MainLayout. AC1
Header.jsx "Legal" link added after "Contact" in both desktop and mobile menus, with aria-label for accessibility. AC2, AC5
Footer.jsx "Aviso Legal / Legal Notice" link destination changed from /terms to /legal. AC4
i18n/locales/es.json Added header.legal and legalPage.* keys in Spanish. AC3
