Database Documentation

MongoDB structure, initialization flow, collection contracts, indexes, and seed data used by the application.

Runtime setup

  • Engine: MongoDB 7 (dockerized for local full-stack mode).
  • Default database: tucreditoonline.
  • Initialization script: database/init/init-db.js.
  • The init script runs automatically the first time the MongoDB container is created.

Collections and schema rules

CollectionMain required fieldsNotes
usersemail, createdAtUses soft delete field deletedAt and optional updatedAt.
credittypesname, baseInterestRate, maxAmount, createdAtCatalog for available loan products and rules used by calculator and request flow.
creditrequestsemail, requestedAmount, status, createdAtStatus enum in validator: Pending, Approved, Rejected.
services(none defined in validator)Container for service catalog data used by frontend informational pages.
contactmessages(none defined in validator)Stores contact form submissions and moderation status.

Schema validators are currently configured for core operational collections and can be expanded as domain requirements evolve.

Index strategy

CollectionIndexesReason

credittypesisActive, name (unique), createdAtPublic pages and calculators filter active products frequently.
creditrequestsstatus, email, createdAt (desc)Admin dashboard and request tracking are status/date driven.
contactmessagesstatus, createdAt (desc)Operational inbox views prioritize message status and recency.

Seed data and defaults

  • The init script inserts four default credit types: Personal Credit, Express Credit, Consolidation Credit, and Business Credit.
  • Each seeded credit type includes range and rate controls: minimum/maximum amount, term limits, and base interest rate.
  • The script prints bootstrap login information for the default admin account used in local environments.

Local operations

  • Start services: docker compose up --build
  • Stop services: docker compose down
  • Reset and re-seed database volume: docker compose down -v

Resetting with -v recreates the volume and reruns initialization scripts from scratch.
