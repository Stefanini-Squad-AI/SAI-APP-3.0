frontend-public

Public customer-facing experience for content browsing, credit simulation, form submission, and multilingual information pages.

Technical foundation

  • Framework: React 18 + Vite SPA with route-based pages.
  • Routing: react-router-dom routes defined in frontend/src/App.jsx.
  • Localization: i18next with en, es, and pt strings shared across pages.
  • Backend integration through service clients in frontend/src/services, with demo fallbacks for static preview deployments.

Detailed page catalog (what each page has and what it does)

Path Primary purpose What the page contains Main user actions Main API calls
/ Present brand value and convert visitors into simulation and contact flows. Hero with CTA buttons, about/mision/vision/values cards, services preview cards, stats strip, and location/contact blocks. Navigate to calculator/services pages, explore highlighted product cards, change language from global selector. GET /api/credittypes?isActive=true
/services Explain available credit products and route user to simulation/contact next step. Credit type cards with amount, term, base rate, feature list, requirements, and CTA to calculator or contact advisor. Review products, compare limits/rates, open calculator flow, contact advisor. GET /api/credittypes?isActive=true
/calculator Estimate installment and financing terms before submitting a formal request. Input controls (type, purpose, income, amount, term), computed result cards, breakdown panel, and request wizard trigger. Adjust simulation variables, compare payment outcomes, open request wizard to send application. GET /api/credittypes?isActive=true, POST /api/creditrequests
/about Build trust through company background and differentiators. Company story section, six value/benefit cards, and CTA to calculator. Read institutional narrative, evaluate trust signals, continue to credit simulation. No backend calls.
/faq Resolve common doubts without contacting support. Accordion by category (general, requirements, amounts, process) and support CTA block. Expand/collapse answers, jump to contact page, call support phone. No backend calls.
/contact Capture customer inquiries and open support channel. Validated contact form, contact information cards, support links, embedded map, and office details. Submit inquiry, trigger success/error feedback, call or WhatsApp support options. POST /api/contactmessages
/privacy Disclose data handling and user privacy rights. Five policy sections with update date, plus contact and back-home links. Read legal clauses and navigate to contact/home pages. No backend calls.
/terms Describe contract and usage conditions for platform services. Five terms sections with update date, plus contact and back-home links. Review legal obligations, then navigate to contact/home pages. No backend calls.
* Handle unknown URLs and recover navigation. 404 code, explanatory text, and CTA back to home. Return safely to root route. No backend calls.
