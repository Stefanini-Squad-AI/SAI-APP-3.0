System Overview

Architecture map, module catalog, routes, i18n boundaries, and integration surfaces.

Architecture and domain

  • frontend/: React 18 + Vite + Tailwind CSS single-page application.
  • backend/: ASP.NET Core Web API (.NET 8) with JWT authentication and MongoDB repositories.
  • database/: MongoDB initialization and seed support for local dockerized environments.
  • automation/: quality tooling, including applied-ai-aura functional automation.

Internationalization (strict)

  • Supported runtime locales are strictly: en, es, and pt.
  • Language switch is provided in the header selector and persisted with tco-user-language.
  • Legal tab labels and bodies on /legal follow the active locale using the same JSON resources as standalone /privacy and /terms pages.

Legal information hub (frontend-public)

  • Public route /legal (see frontend/src/App.jsx) renders LegalInformationPage.jsx with an accessible tablist for privacy and terms content.
  • Standalone routes /privacy and /terms remain for deep links and backward compatibility.
  • Navigation entry points: footer link (data-testid footer-link-legal-information) and header link sharing i18n keys footer.legalInformation / header.legalInformation.
  • Automated preview coverage: AURA feature LegalInformation.feature tagged @preview validates route, tabs, footer navigation, and language persistence.
